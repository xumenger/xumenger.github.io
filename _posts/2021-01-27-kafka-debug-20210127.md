* [https://zookeeper.apache.org/releases.html](https://zookeeper.apache.org/releases.html) 下载apache-zookeeper-3.5.9-bin.tar.gz
* [http://kafka.apache.org/](http://kafka.apache.org/) 下载kafka_2.12-2.7.0.tgz

解压后分别拷贝出以下的文件夹：kafka_2.12-2.7.0_9092、zookeeper-3.5.9_2181
 
## Zookeeper 配置

zookeeper-3.5.9_2181 配置：进入conf 目录下，复制zoo_sample.cfg 为zoo.cfg，然后修改其中的内容如下

```
dataDir=/bigdata/data/zookeeper-3.5.9_2181/zkData
dataLogDir=/bigdata/data/zookeeper-3.5.9_2181/zkLog

# the port at which the clients will connect
clientPort=2181
```

进入bin 目录下，双击zkServer.cmd 即可启动Zookeeper 服务

## Kafka 配置

kafka_2.12-2.7.0_9092 配置。到config 文件夹下 打开server.properties 配置文件进行配置

```
broker.id=100
port=9092
listeners=PLAINTEXT://127.0.0.1:9092
log.dirs=/bigdata/data/kafka_2.12-2.7.0_9092/kafka-logs
zookeeper.connect=127.0.0.1:2181
```

在命令行（Windows CMD）执行以下命令，启动Kafka 服务

```bat
D:
cd bigdata\kafka_2.12-2.7.0_9092\bin\windows

kafka-server-start.bat ../../config/server.properties
```

## 启动源码分析

去[http://kafka.apache.org/downloads](http://kafka.apache.org/downloads) 下载对应版本的Kafka-2.7.0 源码，找到kafka.network.SocketServer.createDataPlaneAcceptorsAndProcessors(SocketServer.scala:251)
 
```scala
// kafka.network.SocketServer.scala
 
247  private def endpoints = config.listeners.map(l => l.listenerName -> l).toMap
 
249  private def createDataPlaneAcceptorsAndProcessors(dataProcessorsPerListener: Int,
250                                                    endpoints: Seq[EndPoint]): Unit = {
251    endpoints.foreach { endpoint =>
252      connectionQuotas.addListener(config, endpoint.listenerName)
253      val dataPlaneAcceptor = createAcceptor(endpoint, DataPlaneMetricPrefix)
254      addDataPlaneProcessors(dataPlaneAcceptor, endpoint, dataProcessorsPerListener)
255      dataPlaneAcceptors.put(endpoint, dataPlaneAcceptor)
256      info(s"Created data-plane acceptor and processors for endpoint : ${endpoint.listenerName}")
257    }
258  }
 
 
private def addDataPlaneProcessors(acceptor: Acceptor, endpoint: EndPoint, newProcessorsPerListener: Int): Unit = {
  val listenerName = endpoint.listenerName
  val securityProtocol = endpoint.securityProtocol
  val listenerProcessors = new ArrayBuffer[Processor]()
  for (_ <- 0 until newProcessorsPerListener) {
    val processor = newProcessor(nextProcessorId, dataPlaneRequestChannel, connectionQuotas, listenerName, securityProtocol, memoryPool)
    listenerProcessors += processor
    dataPlaneRequestChannel.addProcessor(processor)
    nextProcessorId += 1
  }
  listenerProcessors.foreach(p => dataPlaneProcessors.put(p.id, p))
  acceptor.addProcessors(listenerProcessors, DataPlaneThreadPrefix)
}
 
// `protected` for test usage
protected[network] def newProcessor(id: Int, requestChannel: RequestChannel, connectionQuotas: ConnectionQuotas, listenerName: ListenerName,
                                    securityProtocol: SecurityProtocol, memoryPool: MemoryPool): Processor = {
  new Processor(id,
    time,
    config.socketRequestMaxBytes,
    requestChannel,
    connectionQuotas,
    config.connectionsMaxIdleMs,
    config.failedAuthenticationDelayMs,
    listenerName,
    securityProtocol,
    config,
    metrics,
    credentialProvider,
    memoryPool,
    logContext,
  )
}
```

## Java 版本问题

最后怀疑是Java 的版本问题导致的（纯粹是根据经验猜测吧）。试着在cmd 中重新设置一下JAVA_HOME
 
启动服务端命令之前先修改一下JAVA_HOME 环境变量

```bat
D:
cd bigdata\kafka_2.12-2.7.0_9092\bin\windows

// 启动之前，在当前cmd 环境中更改JAVA_HOME 环境变量
set JAVA_HOME=C:\Program Files (x86)\Java\jdk1.8.0_201\jre
set PATH=%JAVA_HOME%\bin;%PATH%
 
kafka-server-start.bat ../../config/server.properties
```

启动成功后，试着创建一个topic，创建成功

```bat
D:
cd bigdata\kafka_2.12-2.7.0_9092\bin\windows
 
set JAVA_HOME=C:\Program Files (x86)\Java\jdk1.8.0_201\jre
set PATH=%JAVA_HOME%\bin;%PATH%
 
kafka-topics.bat --create --zookeeper localhost:2181 --replication-factor 1 --partitions 1 --topic RDP_ARTB_SZ_DCDETAIL_PWD_SZ
```
 
果然没有问题了，因为Kafka 是使用Scala 开发的，所以研究一下Scala 与Java 的版本兼容关系还是很有必要的！

## Java 版本说明
 
看一下这个版本的Java 的信息
 
```
$ java -version

java version "1.8.0_201"
Java(TM) SE Runtime Environment (build 1.8.0_201-b09)
Java HotSpot(TM) Client VM (build 25.201-b09, mixed mode)
```
 
JVM 是32 位的，因为如果是64 位的，在java -version 的输出信息中会有64 Bit 字样
 
查看kafka-server-start.bat 脚本的内容如下，因为操作系统是64 位的，所以会在启动JVM 时设置1024M 的堆，但是可能因为当前机器的运行情况，无法分配这么大的堆，会报错，可以直接修改这个脚本中的堆大小的参数，再重新执行脚本启动Kafka 服务端即可
 
```shell
SetLocal
IF ["%KAFKA_LOG4J_OPTS%"] EQU [""] (
    set KAFKA_LOG4J_OPTS=-Dlog4j.configuration=file:%~dp0../../config/log4j.properties
)
IF ["%KAFKA_HEAP_OPTS%"] EQU [""] (
    rem detect OS architecture
    wmic os get osarchitecture | find /i "32-bit" >nul 2>&1
    IF NOT ERRORLEVEL 1 (
        rem 32-bit OS
        set KAFKA_HEAP_OPTS=-Xmx512M -Xms512M
    ) ELSE (
        rem 64-bit OS
        set KAFKA_HEAP_OPTS=-Xmx1024M -Xms1024M
    )
)
"%~dp0kafka-run-class.bat" kafka.Kafka %*
EndLocal
```
